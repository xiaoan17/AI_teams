# AI Teams Logo Exploration V2

This pass intentionally moves away from literal "feature icons" and toward brand symbols.

Design constraints:

- heavier primary mass, fewer small strokes
- negative-space cuts instead of explanatory line art
- `currentColor` symbol SVGs for theme flexibility
- app-icon presentations with material depth, noise, and restrained amber/focus colors
- wordmark lockups that feel like a developer tool, not an AI landing page

## Directions

| File | Direction | Why it is stronger |
| --- | --- | --- |
| `01-aperture-router` | Command routing aperture | The best primary mark: dense, memorable, and directly tied to routing without drawing terminals. |
| `02-command-monad` | Fused team form | More organic and AI-native; good if the product should feel like "one team" instead of "many panes". |
| `03-context-fold` | Handoff context object | Premium document/product-system signal; stronger for Docs + Git as context. |
| `04-worktree-prism` | Parallel worktree prism | More technical and infrastructure-like, with Git branching encoded as negative space. |
| `05-terminal-sigil` | Custom AT terminal sigil | The closest to a monogram. More ownable, less literal, good for a durable app icon. |
| `06-signal-field` | Focus/status field | Best match for "watch the agents, intervene when needed"; iconic at small sizes. |

Recommended shortlist for refinement:

1. `01-aperture-router`
2. `05-terminal-sigil`
3. `03-context-fold`

Use the app-icon PNGs for feel. Use the raw SVGs for actual implementation.
