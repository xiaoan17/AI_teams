# 第一段 · 设计定调

## 1. 一句话目标

把 AI Teams 从「一个工程内核很硬、但只有熟手能用」的工具，打磨成「**新用户打开就懂、Mac 用户用着顺手**」的产品。
不动已经验证过的底层（真·TUI 渲染、快照恢复、tmux 编排、僵尸回收），只动**用户能感知的那一层**：菜单、首屏、文案、反馈、图标。

## 2. 问题诊断（按用户打开 App 的时间顺序）

| 阶段 | 现状 | 痛点 | 对应工作流 |
|------|------|------|-----------|
| 启动前 | 无原生菜单栏，功能塞在 sidebar 的 `⚙` 折叠里 | `Cmd+Q/W/,` 失效；无法发现「打开日志/切项目/重启」；`ui-open-logs-dir-todo` 一直挂着 | **WS-A** |
| 首屏 | 空状态只有一行字 | 没装 tmux / agent CLI 的新用户直接撞墙，`detectAgents` 能力没在首屏暴露 | **WS-B** |
| 组队 | 每行两个并排 `<select>`（Role + Agent type）+ 状态点 + 三个按钮 | 信息密度过高；新用户分不清 Role vs Agent；占位符暴露内部 id | **WS-C** |
| 全程 | 所有反馈走一个 `notice`，要点击才消失；图标用裸字母 `x`/`+`/`<` | 错误/成功/警告不分级；视觉粗糙；中英混用 | **WS-D** |

## 3. 设计原则

1. **不破坏内核**：WS-A~D 全部是「外观层 + 主进程菜单层」改动，不碰 `tmux-runtime.cjs`、`terminal-wheel.mjs`、快照恢复逻辑。
2. **复用已有能力**：健康检查页直接用 `detectAgents()`；菜单项复用现成 IPC（`shell:openPath`、`workspace:choose`、`agents:stopAll`…）。
3. **以 `RoleConfigModal` 为交互模板**：它的 dirty 守卫、Esc 关闭、分 tab 结构已经打磨得很好，新增的 Settings / 健康检查弹窗与之同构。
4. **i18n 可回退**：缺翻译键时回退到中文（主语言），不留空白。

```

  ✓ 成功   绿色，3s 自动消失
  ℹ 信息   蓝色，5s 自动消失
  右下角 toast 堆叠，不再占据顶部一整条

图标统一（裸字母 → 语义字符/SVG）：
  停止  x → ✕      最小化 - → –      展开 + → +

```

## 5. 取舍与不做的事

| 决策 | 结论 | 理由 |
|------|------|------|
| i18n 方案 | **轻量字典 + Context**，不引第三方库（react-i18next 等） | 当前文案量小（约 60~80 条），自建 `t(key)` + `LocaleContext` 足够，零依赖、可控 |
| 默认语言 | **中文**；缺键回退中文 | 目标用户中文为主；英文作为可切换项 |
| toast 库 | **自建**，不引依赖 | 与零依赖原则一致，复用现有 CSS 变量主题 |
| 图标 | **优先 Unicode 语义字符**，复杂图标才内联 SVG | 避免引入图标库/字体文件，保持包体小 |
| 组队选择器（设计文档 §6.5 的弹出式） | **本计划不强做**，仅在 WS-C 标注「待确认是否补」 | 当前 `startEnabled` 直接批量起，与文档有偏差；属独立议题，不混入本轮 |
| Windows/Linux 菜单 | **本轮只做 macOS**，但菜单模板用跨平台写法预留 | 当前只发 macOS arm64；菜单 role 用 Electron 标准 role 自动适配 |

> 约定：每个任务标注「改动文件 / 关键锚点 / 做什么 / 验收 / smoke」。锚点行号为撰写时快照，实施时以 `grep` 为准。

## 6. WS-A —— 原生菜单栏 + 快捷键

**目标**：新增 Electron `Menu`，把散落操作收进原生菜单，消化 `ui-open-logs-dir-todo`。

### A1. 新建菜单模块
- **改动文件**：新建 `src/main/app-menu.cjs`；`src/main/main.cjs` 引入并在 `app.whenReady` 后 `Menu.setApplicationMenu(buildMenu(...))`。
- **关键锚点**：`main.cjs:1` 已 `require("electron")` 含 `app/BrowserWindow/shell`；需补 `Menu`。`main.cjs:3931` 创建 `mainWindow`。
- **做什么**：`buildMenu({ mainWindow, sendToRenderer, logsDir })` 返回 `Menu.buildFromTemplate(...)`，结构见 §4.1。菜单项分两类：
  - **主进程直接执行**：打开日志目录（`shell.openPath`）、退出、重载、切换全屏。
  - **转发给 renderer**：切换侧边栏、主题、全部启动/停止、配置 Agent、健康检查 —— 通过 `mainWindow.webContents.send("menu:command", { id })` 下发。
- **验收**：菜单栏出现且各项可点；`Cmd+Q/W/R` 生效。
- **smoke**：新增 `scripts/app-menu-smoke.cjs` —— require 模块、传入 mock，断言模板含预期 id、accelerator、`role` 正确，且「打开日志目录」item 的 click 调用了注入的 `openPath`。

### A2. renderer 接收菜单命令
- **改动文件**：`src/main/preload.cjs`（新增 `onMenuCommand`）、`src/renderer/App.jsx`（订阅并分发）。
- **关键锚点**：`preload.cjs:39-65` 现有 `onAgentData` 等 `ipcRenderer.on` 模式，照抄新增 `menu:command`。`App.jsx:2133` 的主 `useEffect` 里挂订阅。
- **做什么**：`onMenuCommand((cmd) => …)` 把 `toggleSidebar / theme:set / effects:toggle / agents:startAll / agents:stopAll / role:configure / onboarding:open` 路由到已有的 `setSidebarCollapsed / setThemeId / setEffectsEnabled / startEnabled / stopEnabled / openRoleConfig / setOnboardingOpen`。
- **验收**：从菜单点「全部启动」「切换侧边栏」「配置 Agent」行为与 sidebar 按钮一致。
- **smoke**：扩展 A1 的 smoke 或在 `App.jsx` 单测层断言命令分发表覆盖所有菜单 id。

### A3. 日志目录入口（消化 TODO）
- **改动文件**：`src/main/main.cjs`（新增 `system:logsDir` 或直接在菜单 click 内拼路径）。
- **关键锚点**：`logger.cjs`（212 行）应已有日志落盘路径；`main.cjs:4072` 已有 `shell:openPath`。
- **做什么**：菜单「打开日志目录」→ `shell.openPath(app.getPath("logs"))` 或 logger 暴露的目录。
- **验收**：点击在 Finder 打开日志目录。
- **收尾**：删除/更新记忆 `ui-open-logs-dir-todo`。

## 8. WS-C —— i18n + 交互主次层级

### C1. i18n 框架（轻量自建）
- **改动文件**：新建 `src/renderer/i18n.js`（`zh`/`en` 字典 + `t(key, vars)` + `LocaleContext`）；`App.jsx` 顶层包 `LocaleProvider`。
- **做什么**：
  - 字典结构 `{ zh: {...}, en: {...} }`，键如 `team.title` / `composer.placeholder` / `agent.start`。
  - `t(key)` 缺键回退 `zh[key]`，再缺回退 key 本身。
  - 语言存 `localStorage` `aiTeams.locale`，默认 `zh`；切换入口放 Settings（WS-A 的偏好设置）+ 菜单「视图 ▸ 语言」。
- **关键锚点**：现有混用文案散落 `App.jsx`（`Team`/`Docs`/`Start`/`Stop`/`Search docs`/`Needs Input`/`配置 Agent`/`已雇`…）、`statusLabels`（`App.jsx:248`）。
- **迁移策略**：**分批**替换字符串为 `t(...)`，先 sidebar + composer + 状态标签，再 modal。每批一个 commit。
- **验收**：切换 zh/en 全 UI 文案随之变；刷新保持；缺键回退中文不留空。
- **smoke**：新增 `scripts/i18n-smoke.cjs` —— 断言 `en` 与 `zh` 键集合一致（无漏译）、`t` 回退逻辑正确。

### C2. Agent 行主次重排
- **改动文件**：`src/renderer/App.jsx`（`Sidebar` 内 agent-row，约 `App.jsx:1521-1634`）、`styles.css`（`.agent-role-select`/`.agent-type-select`）。
- **做什么**：Role select 为主（占满主行）；Agent type 降级为小字副行或 hover/点开展开（§4.3）。不改 `assignRole`/`assignType` 逻辑，只改布局与层级。
- **验收**：默认只见 Role；运行时切换仍可达；键盘可达性（现有 `stopPropagation` 守卫）保留。
- **smoke**：目视 + 现有 `role-config-smoke` 不回归。

### C3. 占位符 / 显示名一致
- **改动文件**：`App.jsx` Composer（`App.jsx:1899` placeholder）。
- **做什么**：placeholder 用 `agentDisplayName`（已存在，`App.jsx:276`）而非裸 id。
- **验收**：占位符显示「@设计师 …」而非「@codex …」。

## 9. WS-D —— 视觉打磨（图标 / toast / 空状态）

### D2. 图标统一

- **验收**：空态可一键进入下一步，不再是死胡同。

## 10. 里程碑与推进顺序

> 原则：**独立可验证、独立可止损**；每个 WS 独立 commit + smoke 绿后再进下一个。

| 顺序 | 里程碑 | 内容 | ROI | 风险 |
|------|--------|------|-----|------|
| **M1** | 原生菜单栏 | WS-A 全部（含消化 open-logs TODO） | 最高（最低成本换「真 Mac App」感） | 低（纯新增） |
| **M2** | 首屏健康检查 | WS-B 全部 | 高（堵新用户流失） | 低（复用 detectAgents） |
| **M3** | 交互降负 | WS-C（i18n 分批 + 主次重排 + 占位符） | 中高 | 中（文案迁移面广，分批降险） |
| **M4** | 视觉打磨 | WS-D（toast + 图标 + 空态 CTA） | 中 | 低 |

### 全局验收（每个里程碑后跑）
```bash
npm run release:check       # 配置卫生 + build + doctor
npm run smoke               # 现有 smoke 全绿（不回归）
# 新增：npm run smoke:app-menu / smoke:i18n / smoke:toast 视该 WS 而定
```

### 收尾动作
- 更新 `README.md`：菜单栏与快捷键说明、首屏健康检查截图。
- 更新记忆：销账 `ui-open-logs-dir-todo`；如菜单/快捷键有踩坑（如 `Cmd+.` 与终端 SIGINT 冲突）则新增一条 feedback 记忆。
- 本文 `status` 推进 `discussion → decided → implementing → done`。

### ⚠️ 已知需提前确认的冲突点
1. **`Cmd+.`（全部停止）** 可能与终端里 CLI 的中断语义冲突——菜单加速键会被 Electron 全局捕获，可能拦掉 xterm 内的 `Cmd+.`。实施 A1 时先验证，必要时换 `Cmd+Shift+.`。
2. **`Cmd+R`（重载）** 在生产构建里应禁用或仅 dev 启用，避免用户误刷丢终端会话——虽然有快照恢复，但仍是惊吓。建议 packaged 态移除 `reload`。
3. **i18n 迁移** 不要一次性全替换；按 §C1 分批，每批跑一次 build 确认无遗漏字符串导致的渲染空白。
